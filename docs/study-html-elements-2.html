2. @media (prefers-color-scheme: dark) — CSS에서 OS 감지

이 박스는 OS 다크모드 설정에 따라 CSS만으로 색이 바뀝니다 (JS 없음)
💡 JS 없이 CSS만으로 OS 다크모드에 반응합니다.
단점: JS로 제어가 불가능 — 사용자가 "OS는 다크지만 이 사이트는 라이트로 보고 싶다"는 요구를 만족할 수 없습니다.
그래서 이 프로젝트는 class 전략 (JS 제어)을 선택했습니다.

4. CSS 우선순위 (Specificity) — :where() / !important

나는 파란색이어야 할까? 빨간색? (태그 vs :where 경쟁)

나는 초록색 (.win 클래스가 태그보다 점수 높음)

나는 !important 로 강제로 주황색

선택자별 점수표:

선택자 점수 예시
인라인 style 1000 style="color:red"
ID 100 #foo
클래스 / 속성 10 .foo, [type]
태그 1 div, pre
:where() 0 :where(pre)
💡 :where(pre)는 태그 선택자지만 점수가 0이라서 어떤 규칙이든 쉽게 덮어쓸 수 있습니다.
Tailwind가 기본 스타일에 :where()를 쓰는 이유입니다.
!important는 점수 계산을 무시하므로, 덮어쓰려면 더 높은 !important가 필요합니다 — 남발 금지.

5. 두 JS 파일 충돌 — shell.js vs reader-controls.js

헤더 (shell.js가 제어 — .dark 클래스 방식)
본문 (reader-controls.js가 제어 — data-theme 방식)

💡 두 버튼이 서로를 모릅니다. 헤더만 바꾸면 본문은 그대로 — 이게 통합 전 문제입니다.
해결: shell.js의 applyTheme() 하나에서 .dark와 data-theme을 동시에 설정.
통합 후 코드 (b.md 섹션 2)

function applyTheme(theme) {
  var isDark = (theme === 'dark') || (theme === 'system' && mql.matches);
  document.documentElement.classList.toggle('dark', isDark); // Tailwind용
  document.documentElement.setAttribute('data-theme', isDark ? 'dark' : 'light'); // 커스텀 CSS용
}
